## Highlight of the Week

There is this [brilliant short video](https://youtu.be/UabBYexBD4k?si=rllX9iWPKDNnkFhb) from IBM, which explains RAG for LLMs and also discusses whether RAG is even still needed in the age of context windows that span a million tokens, which would allow for all the Lord of the Rings books to be added to the prompt.   
Here are some of my notes:  
- RAG can have silent misses (Information exists in source material, but retrieval fails)  
- RAG cannot answer gap questions (identify information that is missing in one document but appears in another)  
- long context doesn't have these issues, but...  
- A growing context dilutes attention, which can lead to needle-in-the-hay-stack issues  
- long context is expensive as you put many tokens into each prompt  
- delivers nice use cases and recommendations
